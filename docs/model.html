Model architecture

A deterministic kernel for coupled infrastructure decisions.

The engine validates scenarios, expands the time axis, computes equation modules, and returns outputs that can be inspected by engineers, executives, analysts, and reviewers.

Input

Scenario schema

Required sections keep each case explicit: metadata, time, phase, energy, cooling, water, manufacturing, economics, governance, policy, and controls.

Kernel

Equation modules

Power, cooling, water, thermodynamics, exergy, manufacturing, economics, governance, policy, and gates run as inspectable modules.

Output

Decision records

Summary metrics, time series, matrix outputs, gate results, reports, and reproducibility hashes make the calculation reviewable.

Equations that matter first

The model turns facility-scale pressure into board-scale consequences.

The implementation is in the repository; this page highlights the mental model for readers evaluating the package.

Thermal removal

Q_remove = P_IT + P_process + P_aux − W_useful − ΔU

Connects electrical and process load to heat rejection.

Exergy destruction

X_destroyed = T0 × S_gen

Shows lost work potential caused by irreversibility.

Water coupling

m_water = Q_remove / (c_p × ΔT × η_cooling)

Links heat removal to withdrawal and consumption pressure.

Net value exposure

V_net = Y × V_gross − C_energy − C_water − C_delay − C_risk

Connects readiness and infrastructure constraints to economics.
