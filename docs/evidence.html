Evidence quality

Good scenarios start with honest labels.

Evidence status tells readers how to treat each input before they look at outputs. That is the difference between a model users can audit and a spreadsheet users have to trust.

Verified

Use when a source is admitted into the repository as a supportable public fact.

Filed

Use when a claim is traceable to a formal public filing or record.

Reported

Use when a public report attributes the claim but the project has not independently verified it.

Assumed

Use when the value is declared to test consequences or stress a boundary.

Evidence flow

Source quality follows the input into the result.

Input

Each material variable carries value, unit, status, source reference, confidence, and notes.

Audit

The schema and evidence audit catch missing fields and invalid statuses.

Output

Reports preserve assumptions, evidence-status summaries, warning flags, and reproducibility hashes.

Source hygiene: keep source URL, source type, publication date, retrieval date, confidence, and impacted model variable together. Use public monitoring examples as templates.
