Deploy the package

Run Terafab Decision Twin from a terminal, a notebook, or a scenario-analysis workflow.

Install terafab-decision-twin, validate evidence-coded scenarios, simulate the deterministic model, and then expand into Monte Carlo, stakeholder strategy, reduced-order trajectories, and validation-readiness analysis.

1. Install

Create a Python 3.10+ environment and install the package from the repository with python -m pip install . or editable mode with python -m pip install -e ..

2. Validate

Run terafab scenario validate scenarios/baseline_2026.json before simulation so schema, evidence labels, and unknown-value discipline are checked first.

3. Analyze

Use CLI exports for reproducible bundles and Python notebooks when you need iterative scenario analysis, uncertainty propagation, or stakeholder decision surfaces.

CLI deployment

Use the command line for repeatable validation, simulation, gates, reports, and export bundles.

The CLI entry point is terafab. It is the fastest path for batch runs, CI checks, and creating auditable JSON, Markdown, and CSV artifacts from declared scenario assumptions.

Create and activate an environment
python -m venv .venv
. .venv/bin/activate

On Windows, activate with .venv\Scripts\activate.

Install the package
python -m pip install -e .
terafab version

Editable installation is recommended while adapting scenarios or docs. Use python -m pip install . for a regular local install.

Validate and simulate a scenario
terafab scenario validate scenarios/baseline_2026.json
terafab simulate scenarios/baseline_2026.json --output runs/baseline.json --report runs/baseline.md

Simulation produces machine-readable results and a decision report from the same scenario file.

Inspect gates and export a bundle
terafab gates scenarios/baseline_2026.json
terafab export scenarios/baseline_2026.json runs/baseline_bundle

Export bundles include result JSON, report Markdown, summary CSV, and gate matrix CSV outputs.

Notebook deployment

Use Python notebooks for exploration, comparison, and stakeholder-facing analysis.

Notebook workflows call the same deterministic kernel as the CLI, which keeps exploratory work aligned with validated scenario files and repeatable outputs.

Install a notebook kernel
python -m pip install -e . notebook
python -m notebook

Open an existing notebook in notebooks/ or create a new one in the activated environment.

Run the deterministic API
from terafab_decision_twin import load_scenario, validate_scenario, run_scenario

scenario = load_scenario("scenarios/baseline_2026.json")
errors = validate_scenario(scenario)
assert not errors, errors
result = run_scenario(scenario)

This mirrors the CLI path: load, validate, run, inspect outputs, and only then interpret scenario consequences.

Add advanced simulation overlays
import json
from pathlib import Path
from strategic_simulation import run_monte_carlo

config = json.loads(Path("strategic_simulation/examples/uncertainty_baseline_2026.json").read_text())
mc = run_monte_carlo("scenarios/baseline_2026.json", config, retain_runs=False)

Use the advanced overlay for declared uncertainty ranges, gate-failure probabilities, metric quantiles, stakeholder strategy analysis, reduced-order trajectories, and decision surfaces.

Start scenario analysis

Treat deployment as a disciplined path from assumptions to auditable decisions.

Choose a scenario boundary

Start with scenarios/baseline_2026.json for a reference run, then copy it when exploring a board case, public-policy watch case, engineering stress test, or investment diligence case.

Label evidence before changing numbers

Every material assumption should carry an evidence status. Unknowns and low-confidence values should be visible rather than hidden in the result.

Run gates before recommendations

Interpret power, cooling, water, wastewater, manufacturing, economics, governance, and policy gates before summarizing a modeled action such as advance, hold, redesign, stage, or monitor.

Escalate from deterministic to strategic analysis

After a deterministic run is stable, use Monte Carlo for uncertainty, game-theory tools for declared stakeholder payoffs, reduced-order models for trajectory screening, and validation-lab reports for readiness gaps.

Deployment boundary: this package deploys an evidence-gated decision model, not verified Terafab operating data. Scenario outputs are consequences of declared assumptions and should not be presented as official Terafab status, private calibration, permitting forecasts, investment advice, or real-world measurements without independent validation.
