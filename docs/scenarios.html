Scenario discipline

Scenarios are instruments for thinking, not claims of private Terafab truth.

A scenario declares a horizon, inputs, evidence labels, allowed actions, and controls. The model then calculates consequences and gate outcomes.

Baseline

A reference case for verifying equations, outputs, and report structure.

Stress test

An assumption-coded exploration of extreme power, water, heat, or policy pressure.

Worst case

A severe-risk pathway used to expose failure boundaries, not to forecast certainty.

Scenario object

Every scenario must be interpretable before it is persuasive.

Metadata

Scenario ID, author, model version, source bundle, purpose, and type.

Material inputs

Energy, cooling, water, manufacturing, economics, governance, and policy variables.

Controls

Strict evidence mode, unknown substitution rules, output registry, and allowed actions.

Run included scenarios

Use the README workflow before interpreting any scenario output.

Validation comes first so schema, evidence labels, and unknown-value discipline are checked before simulation.

Validate a scenario
terafab scenario validate scenarios/baseline_2026.json
Simulate and write a report
terafab simulate scenarios/baseline_2026.json --output runs/baseline.json --report runs/baseline.md
Compare gates or export a bundle
terafab gates scenarios/baseline_2026.json
terafab export scenarios/baseline_2026.json runs/baseline_bundle

Interpretation rule: numerical results are scenario-dependent outputs. They should never be summarized as real-world measurements unless independently validated.
