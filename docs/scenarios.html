Scenario guide

Scenarios are the working language of the package.

Each scenario declares assumptions, evidence labels, controls, and a time horizon. The model turns that declared case into outputs, reports, and gate results.

Included scenarios

Start with the cases already in the repository.

Use these files to understand the range of baseline, stress, multi-year, upside, and downside thinking supported by the package.

Create a scenario

Use the CLI when you want a clean starting point.

terafab scenario new my_case scenarios/my_case.jsonterafab scenario validate scenarios/my_case.jsonterafab simulate scenarios/my_case.json --output runs/my_case.json --report runs/my_case.md

Material sections

Energy, cooling, water, manufacturing, economics, governance, and policy are separate so each assumption can be reviewed.

Evidence fields

Use value, unit, status, source_ref, confidence, and notes on material inputs.

Outputs

Inspect scalar summaries, time series, gate matrix, reports, and warning flags before making comparisons.

Tip: Use the public monitoring examples when a scenario starts from public filings, public records, reported claims, or user-provided assumptions.
