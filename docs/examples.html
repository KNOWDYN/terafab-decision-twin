Public monitoring playbook

Turn public signals into reviewable scenario inputs.

The root /examples directory shows how boards, policy teams, engineers, investors, and researchers can map public information into evidence-coded inputs.

Collect

Public signals

Capture public filings, agency records, utility disclosures, water authority records, reported claims, and user assumptions.

Classify

Evidence status

Use the narrowest honest status before a value enters the scenario.

Compute

Decision outputs

Run validation, simulate the scenario, and inspect outputs and gates.

Traceability path

Source claim → evidence status → model variable → output → decision gate.

Start with a checklist

Use examples/public_monitoring_checklist.md to decide what can be monitored.

Record the source

Use the evidence intake and source register templates.

Run a scenario

Use the progress-watch or infrastructure-watch scenario as a schema-valid starting point.
