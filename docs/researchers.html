Researcher guide

Reproducible public modeling for technical reviewers and human researchers.

The repository is designed so analysts can inspect the schema, run examples, check gates, audit equations, and extend modules without relying on hidden data.

Repository map

  • schema/scenario_schema.json — scenario input contract.
  • scenarios/ — baseline, stress-test, and multi-year examples.
  • terafab_decision_twin/models/ — implemented equation modules.
  • sources/ — admitted facts and source governance files.
  • tests/ — schema, CLI, equation, evidence, and engine checks.

Citation posture

Cite both the GitHub repository and this Pages site. When discussing scenario results, cite the scenario file and evidence status.

Review checklist

Reproducibility

Can the scenario be validated and run from the CLI?

Evidence

Are assumptions clearly separated from verified facts?

Physics

Do conservation and unit checks pass?

Policy interpretation

Are outputs framed as conditional decision support?

Terafab Decision Twin

Public, evidence-gated decision support for Terafab-scale semiconductor energy, water, exergy, yield, economic, governance, and policy analysis.

Boundaries

No Terafab endorsement is claimed. Scenario assumptions are not verified operating facts. Restricted sources are not redistributed.
